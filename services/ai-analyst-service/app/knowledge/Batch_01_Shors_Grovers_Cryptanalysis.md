### Q: How does Shor's algorithm find the prime factors of an integer? ### Q: How does Grover's algorithm search unstructured databases in quadratic time? A: Grover's algorithm solves the unstructured search problem, finding a marked item among N possibilities, in O(√N) queries to an oracle function, compared to the O(N) queries required classically in the worst case. It achieves this through amplitude amplification, a process where the algorithm iteratively applies a combination of an oracle (which marks the target state by flipping its phase) and a diffusion operator (which reflects amplitudes about the average), progressively increasing the probability amplitude of the marked state while decreasing the amplitudes of all other states. After approximately (π/4)√N iterations, measuring the quantum state yields the target item with high probability. This quadratic speedup is provably optimal for unstructured search; no quantum algorithm can solve this problem faster than O(√N), which is a meaningful constraint compared to Shor's algorithm's exponential speedup. In the context of cryptography, this algorithm is applied to brute-force key search problems, where the oracle checks whether a candidate key correctly decrypts a known plaintext-ciphertext pair, effectively halving the perceived bit-strength of any symmetric cipher or hash function against a quantum adversary using this technique.

### Q: What is the difference between pre-image, second pre-image, and collision attacks under Grover's search? A: A preimage attack attempts to find any input that hashes to a specific given output, modeled directly as Grover's unstructured search over the input space, reducing an n-bit hash's preimage resistance from 2^n classically to 2^(n/2) under quantum attack. A second-preimage attack attempts to find a different input that produces the same hash output as a specific known input, which is structurally similar to a preimage attack and receives the same quadratic speedup under Grover's algorithm, reducing resistance from 2^n to 2^(n/2) as well. A collision attack, by contrast, attempts to find any two inputs that produce the same hash output without either being specified in advance, which classically already benefits from the birthday paradox, achieving 2^(n/2) complexity rather than 2^n; under quantum attack using algorithms like the BHT algorithm, this can be further reduced to approximately 2^(n/3), though with significant quantum memory overhead that limits practical feasibility. This distinction matters for cryptographic protocol design because applications relying on collision resistance, such as digital signature schemes that hash a message before signing, require larger hash outputs to maintain equivalent security margins compared to applications relying only on preimage resistance, such as password hashing or commitment schemes, which is why NIST recommends hash output lengths of at least 256 bits for general-purpose post-quantum security across all three attack categories.
