### Q: What is the Module Learning With Errors (M-LWE) problem? A: The Module Learning With Errors problem is the core hardness assumption underlying ML-KEM (Kyber), generalizing the standard Learning With Errors (LWE) problem by working over modules constructed from a polynomial ring rather than over plain integer vectors, which allows for smaller key sizes while maintaining comparable security. The "module" structure specifically uses polynomial rings such as Z_q[x]/(x^n + 1), which allows operations to be expressed efficiently using Number Theoretic Transform-based polynomial multiplication, striking a balance between the pure, less structured (and more conservative) Learning With Errors problem and the more structured, higher-performance Ring-LWE problem. ### Q: How does ML-KEM prevent chosen-ciphertext attacks (CCA)? A: ML-KEM achieves IND-CCA2 security, meaning resistance against adaptive chosen-ciphertext attacks, through the application of a variant of the Fujisaki-Okamoto (FO) transform, which converts the underlying weaker IND-CPA-secure (chosen-plaintext attack secure only) public-key encryption scheme into a fully IND-CCA2-secure key encapsulation mechanism. The core technique involves re-encrypting the decrypted message during decapsulation and comparing it against the received ciphertext; if they do not match exactly, the decapsulation procedure returns a pseudorandom value derived from a hash function rather than an error message or the actual decrypted content, which prevents an attacker from learning anything about the secret key by submitting maliciously crafted ciphertexts and observing how the system responds to invalid inputs. This implicit rejection mechanism, where invalid ciphertexts produce indistinguishable pseudorandom output rather than explicit error signals, is critical because explicit error signaling in earlier lattice-based encryption schemes has historically enabled chosen-ciphertext attacks that could leak secret key information bit by bit through carefully constructed invalid inputs, a class of attack that affected several pre-NIST-competition lattice scheme designs. ML-KEM's specific implementation of this FO-transform variant has undergone extensive cryptanalytic review during the multi-year NIST PQC standardization process, with no practical chosen-ciphertext attacks identified against properly implemented ML-KEM, making correct implementation of the implicit rejection logic (rather than the mathematical scheme itself) the primary security-critical concern for engineering teams deploying ML-KEM.

### Q: Is ML-KEM vulnerable to classical lattice reduction attacks like Lenstra–Lenstra–Lovász (LLL)? A: ML-KEM's security parameters were specifically selected to resist known classical lattice reduction attacks, including the LLL algorithm and its more powerful successor, the BKZ (Block Korkine-Zolotarev) algorithm, which remains the most effective known classical approach to solving the underlying Shortest Vector Problem that ML-KEM's security rests upon, but neither algorithm comes close to threatening ML-KEM's security margin given the lattice dimensions used in the standardized parameter sets.
